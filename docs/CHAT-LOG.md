## 7. Production: Vercel + Render + PostgreSQL

**Контекст:** >1000 организаций, много данных, частые изменения логики.

**Архитектурное решение:**
- **Vercel** — портал (статика, Root Directory: `portal`).
- **Render** — API (`server/`).
- **PostgreSQL** — постоянное хранение (данные не теряются при redeploy).

**Настройка:**
- `VITE_API_URL` на Vercel → URL API на Render.
- `DATABASE_URL` на Render → managed PostgreSQL.
- `OKO_BOOTSTRAP_ADMIN_USER` / `OKO_BOOTSTRAP_ADMIN_PASSWORD` — первый admin.

**Исправления PostgreSQL:**
- Пустые даты `""` → `dateOrNull()` (`f097978`).
- `Date` objects из pg (`57dc65d`).
- Поддержка `OkoDb` abstraction (`14b6761`).

**Документация:** [DEPLOY.md](DEPLOY.md).

## 13. Офлайн-обмен с дочерними (июнь 2026)

**Контекст:** филиалы без доступа к порталу ЦО.

**Реализовано (`49d95fa`, `c4b29fb`):**
- `POST /api/packages/import`, движок `packageImport` / `packageExport` v1.1.
- Страница **Сводка и импорт** — выгрузка JSON и **offline-kit** (zip).
- Offline-kit: статический портал + `offline-package.json` + `start.bat` / `start.sh`.
- Документация: [OFFLINE-EXCHANGE.md](OFFLINE-EXCHANGE.md).

**Исправления по ходу эксплуатации:**

| Проблема | Причина | Решение | Коммит |
|----------|---------|---------|--------|
| Кнопки «Скачать offline-kit **(0)**» | Строгое сравнение ZID/EID, legacy-записи без привязки | `loadWorkPackageInstances`, нормализация ID на сервере | `a722930` |
| `Unauthorized` при создании формы в offline-kit | Портал пытался вызвать API (вшитый `VITE_API_URL`) | `initStorage` → только localStorage в offline-режиме | `a26e6a0` |
| Запуск `start.bat` тянет Node.js | Использовался `npx serve` | Встроенный **PowerShell**-сервер (`serve.ps1`) | `1142ee5` |
| Ошибки `'ile'`, `'акрывайте'` в cmd | UTF-8 и кириллица в `.bat` | ASCII-only `start-offline.bat`, CRLF | `4036dac` |
| Браузер не открывается сам | — | `start "" http://localhost:8787/` после паузы | `4036dac` |

**Уточнение терминологии:** «без сервера» = без **сервера ЦО** и интернета; локальный мини-сервер на ПК (PowerShell / Python) нужен браузеру для SPA.

**Локальная разработка API (`a26e6a0`):**
- Загрузка `.env` через `dotenv` в `server/src/env.ts`.
- `OKO_AUTH_DISABLED=1` в `.env.example` — вход не обязателен при dev.
- Подсказки на странице входа и понятные сообщения при 401.
