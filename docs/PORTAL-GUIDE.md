# Инструкция пользователя организации

Руководство по порталу ОКО для представителя организации. Здесь описано **весь доступный вам функционал**: зачем он нужен, как устроен, какие кнопки нажимать и чего ожидать в результате.

Портал заменяет работу в MS Access «ПК ОКО»: вы заполняете **76 форм** корпоративной отчётности, сохраняете их на сервере, проверяете правильность и сдаёте в центральный офис.

## 1. Общие сведения

### Что такое портал ОКО

Это веб-система для подготовки комплекта корпоративной отчётности. В каталоге **76 шаблонов форм** (N01, N02, ND и др.). По каждому шаблону создаётся **экземпляр** — ваша заполненная копия с данными за конкретный отчётный период.

Данные хранятся **на сервере** (не в браузере). Войдя с любого компьютера под своим логином, вы увидите те же формы.

### Ваша роль

Учётная запись с ролью **«Организация»** привязана к одному юридическому лицу. Вы видите и редактируете **только формы своей организации**. Формы других организаций группы вам недоступны.

### Основные понятия

**ZID** — код вашей организации в справочнике системы. Задаётся администратором, менять его вам не нужно.

**EID** — код отчётного периода (квартал, полугодие, год и т.д.). Периоды создаёт администратор; вы выбираете нужный из списка.

**Комплект отчётности** — набор из до **76 экземпляров форм** для пары «организация + период». Это аналог выбора организации и периода в старом Access.

**Рабочий контекст** — текущие ZID и EID, которые вы выбрали на странице **Комплект**. Все новые формы автоматически привязываются к этому периоду.

**Статусы формы:**

| Статус | Значение |
|--------|----------|
| **Черновик** | Форма в работе: можно редактировать, сохранять, удалять, сдавать |
| **Сдано** | Форма зафиксирована: редактирование закрыто, доступен только просмотр и выгрузка |

### Что вам доступно и что нет

**Доступно:** Комплект, Мои формы, Каталог, Настройки, Инструкция.

**Недоступно:** Сводка и импорт (пакетные операции ЦО), редакторы правил, управление пользователями, создание организаций и периодов, просмотр чужих форм.

## 2. Вход в портал и навигация

### Зачем нужен вход

Вход по логину и паролю защищает данные и определяет, к какой организации вы привязаны. Без авторизации портал не покажет ваши формы.

### Как войти

1. Откройте в браузере адрес портала — откроется **главная страница** с формой входа.
2. Введите **логин** и **пароль**, выданные администратором.
3. Нажмите кнопку **Войти**.

**Результат:** откроется раздел **Мои формы**. Внизу бокового меню слева отобразятся ваше имя и название организации — убедитесь, что это правильная учётная запись.

### Как выйти

Внизу бокового меню нажмите **Выйти**.

**Результат:** сессия завершена, снова отображается страница входа.

### Боковое меню — обзор разделов

| Пункт меню | Для чего нужен |
|------------|----------------|
| **Комплект** | Выбор отчётного периода, контроль полноты (сколько из 76 форм заведено и сдано), массовое создание пустых форм |
| **Мои формы** | Список всех ваших сохранённых форм: поиск, открытие, удаление |
| **Каталог** | Просмотр всех 76 шаблонов; создание отдельной формы вручную |
| **Инструкция** | Это руководство |
| **Настройки** | Реквизиты по умолчанию для новых форм; выход из учётной записи |

В правом верхнем углу может отображаться метка вида `POSTGRESQL · user` — это значит, что портал подключён к серверу и данные сохраняются в базе.

## 3. Раздел «Комплект»

**Меню → Комплект**

### Зачем нужен этот раздел

**Комплект** — ваш главный инструмент контроля отчётности за период. Здесь вы:

- выбираете, за какой период работаете;
- видите, сколько форм уже заведено из 76;
- одной кнопкой создаёте все недостающие пустые формы;
- переходите к заполнению конкретной формы.

Без заведения комплекта формы можно создавать по одной из каталога, но удобнее завести весь набор сразу.

### Рабочий контекст: выбор периода

В верхней части страницы блок **«Рабочий контекст»**:

- **Организация (ZID)** — обычно уже выбрана ваша организация, список заблокирован (если организация одна).
- **Период (EID)** — выпадающий список отчётных периодов.

**Как пользоваться:**

1. Откройте **Комплект**.
2. В списке **Период (EID)** выберите нужный отчётный период (например, «1 квартал 2026»).
3. Под списками появится подсказка с названием организации и датами периода.

**Как это работает:** выбор автоматически сохраняется на сервере. Все новые формы, которые вы создадите из каталога, будут относиться к выбранному периоду.

**Результат:** активен рабочий контекст; ниже обновится блок полноты комплекта.

### Полнота комплекта

Блок **«Полнота комплекта»** показывает готовность отчётности:

- счётчик **заведено / 76** — сколько шаблонов уже имеют экземпляр;
- **Черновики** — формы в работе;
- **Сдано** — зафиксированные формы;
- **прогресс-бар** — наглядная доля заведённых форм.

**Как это работает:** система сравнивает полный каталог из 76 шаблонов с вашими сохранёнными экземплярами за выбранный EID.

**Результат:** вы сразу видите, сколько форм ещё нужно завести или сдать.

### Завести пустые формы (комплект)

**Зачем:** создать пустые экземпляры для всех шаблонов, которых ещё нет в комплекте за выбранный период. Это старт работы над новым периодом.

**Как пользоваться:**

1. Убедитесь, что выбраны организация (ZID) и период (EID).
2. Нажмите синюю кнопку **Завести пустые формы (комплект)**.
3. Дождитесь окончания — кнопка покажет «Создание…», затем появится строка статуса.

**Как это работает:** для каждого из 76 шаблонов, у которого ещё нет экземпляра в этом комплекте, создаётся пустая форма с предзаполненными строками показателей из шаблона. Уже существующие формы **не изменяются и не удаляются**.

**Результат:** сообщение вида «Комплект заведён: создано X, пропущено Y (всего шаблонов 76)»; счётчик полноты обновится; формы появятся в **Мои формы**.

### Списки заведённых и незаведённых форм

- **Заведено (N)** — раскрывающийся список. У каждой формы: код, название, статус (черновик/сдано). Клик по коду открывает редактор.
- **Не заведено (N)** — шаблоны без экземпляра. Ссылка ведёт в **Каталог**, где можно создать форму вручную.

Рядом есть ссылка **Мои формы** — переход к полному списку экземпляров.

## 4. Раздел «Мои формы»

**Меню → Мои формы**

### Зачем нужен этот раздел

Здесь хранятся **все ваши сохранённые экземпляры** форм — и черновики, и сданные. Это ваш «личный кабинет» для ежедневной работы: найти форму, открыть, удалить, импортировать резервную копию.

### Поиск и фильтры

В верхней панели:

- **Поиск** — строка «Поиск по названию, коду, организации…»; вводите часть названия или кода формы (например, `N01_1`).
- **Все типы форм** — фильтр по коду шаблона.
- **Все статусы** — «Черновики» или «Сдано».

**Результат:** список карточек сокращается до подходящих записей.

### Карточка формы

На каждой карточке: название, код шаблона, полное наименование формы, статус, организация, период, даты создания и изменения.

**Кнопка «Открыть»** (или клик по названию) — переход в редактор формы.

**Кнопка «Удалить»** — удаление экземпляра. Появится диалог подтверждения; после согласия форма удаляется **безвозвратно**.

### Массовое удаление

Если нужно удалить несколько форм сразу:

1. Отметьте **чекбоксы** слева на карточках нужных форм.
2. Или нажмите **Выбрать все (N)** над списком — отметятся все видимые формы (с учётом фильтра).
3. В панели фильтров появятся кнопки **Удалить выбранные (N)** и **Снять выбор**.
4. Нажмите **Удалить выбранные (N)**, подтвердите в диалоге.

**Результат:** выбранные формы удалены из базы.

### Импорт JSON

**Зачем:** загрузить ранее сохранённую форму из файла (резервная копия или перенос с другого компьютера).

**Как:** нажмите **Импорт JSON** → выберите файл `.json` на диске.

**Результат:** создаётся новый экземпляр, открывается редактор.

## 5. Раздел «Каталог»

**Меню → Каталог**

### Зачем нужен этот раздел

**Каталог** — полный список из **76 шаблонов** корпоративной отчётности. Здесь можно:

- найти нужную форму по коду или названию;
- посмотреть PDF-образец бланка;
- создать **один** экземпляр вручную (в отличие от массового заведения в Комплекте).

### Поиск и фильтрация

- Строка поиска — по коду (`N01_1`) или названию.
- Выпадающий список **Все разделы** — фильтр по разделу (N01, N02, … ND).

### Создание формы из каталога

**Рекомендуется** перед созданием выбрать период в **Комплект**, чтобы форма попала в правильный комплект.

1. Найдите нужный шаблон.
2. Нажмите **Создать** на карточке шаблона.
3. Дождитесь открытия редактора.

**Результат:** новый экземпляр появится в **Мои формы** с привязкой к текущему ZID и EID.

Кнопка **PDF** (если есть) открывает образец пустого бланка в новой вкладке — для сверки структуры при заполнении.

## 6. Редактор формы

Открывается из **Мои формы** или **Комплект** по клику на форму.

### Зачем нужен редактор

Это основное рабочее место: ввод данных в таблицу формы, проверка правильности, сохранение на сервер, сдача, выгрузка в PDF/Excel.

### Элементы экрана

**Верхняя панель:**

- ссылки **← Мои формы ОКО** и **Каталог** — навигация назад;
- поле **Название сохранённой формы** — произвольное имя для списка;
- код формы, полное название, **бейдж статуса** (черновик/сдано);
- **панель кнопок** — все действия с формой.

**Реквизиты** (под панелью):

| Поле | Назначение |
|------|------------|
| Организация | Наименование в шапке формы |
| Код предприятия | Код в формате ОКО (например, `1@1`) |
| Начало / конец периода | Даты отчётного периода |
| Единица измерения | Обычно «тыс.руб.» |
| Подписи | ФИО и должности ответственных лиц |

**Таблица данных** — основное поле ввода. Строки и колонки соответствуют бумажному бланку. Часть ячеек заблокирована (итоги, коды строк) — их нельзя редактировать вручную, они заполняются пересчётом.

### Рекомендуемый порядок работы с формой

1. Проверьте реквизиты (организация, период).
2. Заполните ячейки таблицы.
3. Нажмите **Сохранить**.
4. При необходимости — **Пересчёт** (или включите **Автопересчёт**).
5. Нажмите **Проверить форму** — исправьте ошибки.
6. Снова **Сохранить**.
7. Нажмите **Сдать форму**.

### Описание кнопок

#### Сохранить

**Зачем:** записать все изменения на сервер. Пока не нажали «Сохранить», данные есть только в браузере и могут потеряться при закрытии вкладки.

**Результат:** зелёная строка «Сохранено ЧЧ:ММ:СС»; данные доступны с любого компьютера после входа.

#### Сдать форму

**Зачем:** зафиксировать форму как готовую к приёмке центральным офисом. После сдачи вы не сможете её редактировать.

**Как:** убедитесь, что форма сохранена и проверена → нажмите **Сдать форму** → подтвердите в диалоге.

**Результат:** статус меняется на **«Сдано»**; появляется жёлтая полоса «Форма сдана и доступна только для просмотра»; кнопка «Сохранить» и поля таблицы блокируются.

#### Пересчёт

**Зачем:** пересчитать итоговые строки и графы по правилам шаблона (суммы, производные показатели).

**Когда нажимать:** после ввода чисел в исходные ячейки, если итоги не обновились автоматически.

**Результат:** строка «Строки пересчитаны»; итоговые ячейки обновлены.

#### Автопересчёт (галочка)

**Зачем:** не нажимать «Пересчёт» вручную после каждого ввода — итоги обновляются сами через ~полсекунды.

**Результат:** настройка сохраняется в браузере.

#### Проверить форму

**Зачем:** автоматическая проверка **увязок** — контрольных соотношений между ячейками этой и других форм комплекта (например, сумма в одной форме должна совпадать с другой).

**Как:** нажмите **Проверить форму**; подождите — появится панель результатов.

**Результат:** если ошибок нет — «Проверка пройдена». Если есть — список с указанием формы, ячейки, ожидаемого и фактического значения. Исправьте данные, снова сохраните и проверьте.

#### Проверить расшифровки

**Зачем:** только для форм с контрагентами — проверка расшифровок по справочнику и порогам сумм.

**Результат:** список ошибок и предупреждений или сообщение об отсутствии замечаний.

#### Сохранить PDF

**Зачем:** получить печатную версию заполненной формы с таблицей, реквизитами и подписями.

**Результат:** файл PDF скачивается в папку «Загрузки» браузера.

#### Excel

**Зачем:** выгрузить данные формы в таблицу Excel для анализа или передачи.

**Результат:** файл `.xlsx` скачивается на диск.

#### Экспорт JSON

**Зачем:** полная резервная копия экземпляра (все ячейки, реквизиты, метаданные).

**Результат:** файл `.json`, который можно импортировать через **Мои формы → Импорт JSON**.

#### Импорт

**Зачем:** загрузить данные из JSON-файла **в текущую** форму (заменяет существующие данные).

**Внимание:** используйте осторожно — текущие данные будут перезаписаны.

#### Сбросить данные

**Зачем:** очистить все введённые значения и вернуть форму к пустому шаблону.

**Результат:** после подтверждения все ячейки как при первом создании.

#### Удалить

**Зачем:** удалить экземпляр формы полностью.

**Результат:** после подтверждения — переход в **Мои формы**, форма исчезает из списка.

#### Образец PDF

**Зачем:** открыть пустой бланк формы для сверки при заполнении.

**Результат:** PDF в новой вкладке браузера.

### Если форма уже сдана

Редактирование закрыто. Доступны просмотр и выгрузка (PDF, Excel, JSON). Для исправлений обратитесь к **администратору** — он нажмёт **Вернуть в черновик** и сообщит вам.

## 7. Раздел «Настройки»

**Меню → Настройки**

### Зачем нужен

Здесь задаются **значения по умолчанию** для **новых** форм: код предприятия, текст организации в шапке, даты периода, единица измерения. Уже созданные формы эти настройки не меняют.

### Поля

| Поле | Что подставляется |
|------|-------------------|
| Код предприятия | В реквизиты новых форм |
| Организация | Текст в шапке (не путать со справочником ZID!) |
| Начало / конец периода | Если не задан EID в Комплекте |
| Единица измерения | Например, «тыс.руб.» |

Нажмите **Сохранить настройки** → появится надпись **Сохранено**.

### Учётная запись

Отображаются логин, роль, организация. Кнопка **Выйти** — завершение сессии.

## 8. Полный цикл работы за период

Типовая последовательность от начала периода до сдачи всего комплекта:

### Шаг 1. Подготовка комплекта

1. **Комплект** → выберите период (EID).
2. **Завести пустые формы (комплект)** → дождитесь «76/76».

### Шаг 2. Заполнение

1. **Мои формы** → фильтр «Черновики».
2. Для каждой формы: **Открыть** → заполнить таблицу → **Сохранить** → **Пересчёт** → **Проверить форму** → исправить ошибки → **Сохранить** → **Сдать форму**.

### Шаг 3. Контроль

1. **Комплект** → убедитесь: **76/76** заведено, **Сдано: 76** (или все нужные формы сданы).

**Итог:** комплект за отчётный период заполнен и сдан.

---

## 9. Частые вопросы

**Где мои данные после «Сохранить»?**  
На сервере. Войдите с другого компьютера под тем же логином — форма на месте.

**Могу ли я исправить сданную форму?**  
Нет, самостоятельно. Попросите администратора вернуть форму в черновик.

**Почему нет раздела «Сводка и импорт»?**  
Он только у администраторов ЦО для пакетных операций по всей группе.

**Почему не могу сменить организацию?**  
Ваша учётная запись привязана к одному ZID — это разграничение данных.

**Чем «Сохранить» отличается от «Сдать»?**  
«Сохранить» — промежуточная запись, можно продолжать правки. «Сдать» — финальная фиксация, правки закрыты.

**Много ошибок при «Проверить форму»**  
Ошибки увязок часто связаны с **другими формами** комплекта. Откройте указанные формы, исправьте ячейки, сохраните все затронутые формы и проверьте снова.

## 1. Роль администратора

### Зачем нужна эта роль

Администратор управляет **всей группой организаций**: создаёт справочники, периоды, учётные записи, контролирует полноту и качество отчётности, выполняет пакетные операции (проверка, пересчёт, агрегация, перенос сальдо), настраивает правила увязок и возвращает сданные формы в черновик для исправлений.

### Чем администратор отличается от пользователя организации

| Возможность | Пользователь org | Администратор |
|-------------|:----------------:|:-------------:|
| Заполнять и сдавать свои формы | ✓ | ✓ |
| Заводить комплект форм | ✓ (свой ZID) | ✓ (любой ZID) |
| Создавать организации и периоды | ✗ | ✓ |
| Пакетная проверка / пересчёт | ✗ | ✓ |
| Агрегация и перенос сальдо | ✗ | ✓ |
| Редакторы правил (увязки, сальдо…) | ✗ | ✓ |
| Управление пользователями | ✗ | ✓ |
| Вернуть сданную форму в черновик | ✗ | ✓ |
| Просмотр всех организаций | ✗ | ✓ |

### Вход

1. Откройте главную страницу портала (`/`).
2. Введите логин и пароль администратора.
3. Нажмите **Войти**.

**Результат:** после входа откроется раздел **Комплект** (у пользователя org — **Мои формы**).

## 3. Комплект: организации и периоды

**Меню → Комплект**

### Зачем нужен

Здесь задаётся **инфраструктура отчётности**: какие организации есть в группе, какие у них периоды, и заводятся пустые формы на комплект. Пользователи org видят только свою организацию; администратор может переключаться между всеми ZID.

### Создать организацию

**Когда:** в группу вошло новое юридическое лицо.

1. В блоке **Добавить организацию** введите наименование.
2. Нажмите **Создать организацию**.

**Результат:** сообщение «Организация «…» создана (ZID=N)»; организация появляется в списке **Организация (ZID)**.

### Создать период

**Когда:** начинается новый отчётный период (квартал, год и т.д.).

1. Выберите **Организация (ZID)** — период привязывается к организации.
2. В блоке **Добавить период** заполните название и даты начала/окончания.
3. Нажмите **Создать период**.

**Результат:** «Период «…» создан (EID=N)»; период доступен в списке **Период (EID)**.

> Обычно периоды создают для каждой организации или используют единый EID в рамках группы — по вашей методике.

### Выбор рабочего контекста

1. Выберите **Организация (ZID)**.
2. Выберите **Период (EID)**.

**Как это работает:** контекст сохраняется на сервере. Разделы **Сводка и импорт**, **Мои формы** (фильтр) и новые формы из каталога ориентируются на эту пару.

### Завести пустые формы (комплект)

**Зачем:** создать до 76 пустых экземпляров для выбранной org и периода.

1. Убедитесь, что ZID и EID выбраны.
2. Нажмите **Завести пустые формы (комплект)**.

**Результат:** «Комплект заведён: создано X, пропущено Y»; счётчик полноты обновится; ссылка **Мои формы** ведёт к списку.

Существующие формы не перезаписываются.

### Контроль полноты

Блок **Полнота комплекта**: заведено N/76, черновики, сдано, прогресс-бар. Списки **Заведено** и **Не заведено** — быстрый переход к формам.

## 4. Сводка и импорт

**Меню → Сводка и импорт**

### Зачем нужен этот раздел

Аналог пакетных операций в старом Access «ПК ОКО»: работа **со всем комплектом** выбранного периода сразу — выгрузка, пересчёт, проверка увязок, перенос сальдо, свод по группе.

Страница использует **рабочий контекст** из Комплекта (ZID + EID). Если в периоде 0 форм — большинство кнопок неактивны; сначала заведите комплект.

---

### Блок «Сохранить комплект на диск»

**Зачем:** резервное копирование или передача всего комплекта за период.

#### JSON (N)

**Что делает:** скачивает один файл со всеми формами периода (N — число форм).

**Как:** нажмите **JSON (N)** → файл сохранится в «Загрузки».

**Результат:** архив для хранения или импорта на другой стенд.

#### Excel

**Что делает:** выгружает комплект в один файл Excel.

**Как:** нажмите **Excel**.

**Результат:** файл `.xlsx` со всеми формами периода.

#### Пересчитать все

**Что делает:** запускает пересчёт итоговых строк во **всех** формах периода (как кнопка «Пересчёт» в каждой форме, но пакетно).

**Когда:** после массового импорта или если пользователи жалуются на неверные итоги.

**Как:** нажмите **Пересчитать все** → дождитесь завершения.

**Результат:** «Пересчитано N форм»; итоговые ячейки обновлены и сохранены.

---

### Блок «Проверка форм»

**Зачем:** найти все нарушения **увязок** (контрольных соотношений) по комплекту до дедлайна сдачи.

**Режим проверки** (выпадающий список):

| Режим | Что проверяется |
|-------|-----------------|
| **Период** | Правила, привязанные к текущему периоду |
| **Активные** | Только активные правила |
| **Все правила** | Полный набор правил из справочника |

**Как пользоваться:**

1. Выберите режим.
2. Нажмите **Проверить все формы**.
3. Дождитесь панели результатов (может занять время на большом комплекте).

**Результат:** список нарушений с указанием формы, ячейки, ожидаемого и фактического значения. «0 ошибок» — увязки выполнены. Исправления — в редакторе форм (**Мои формы → Открыть**), затем повторная проверка.

---

### Блок «Перенос сальdo»

**Зачем:** перенести остатки (сальдо) из одного экземпляра формы в другой — например, с прошлого периода на текущий или между этапами отчётности. Аналог операций сальдо в Access.

**Способ** (режим переноса):

- **FormCorrespondence** — по соответствию форм (этапы Yellow/Red).
- **a_tblsaldo** — по правилам из справочника сальдо (типы T/S/G).

**Как пользоваться:**

1. Выберите **Способ**.
2. Укажите **Исходная форма** и **Целевая форма** (оба экземпляра должны быть **одного шаблона**).
3. Для FormCorrespondence — выберите этап (Yellow/Red).
4. Для a_tblsaldo — выберите тип T/S/G.
5. Нажмите **Перенести сальdo**.

**Результат:** значения перенесены в целевую форму и сохранены; строка статуса подтверждает операцию.

---

### Блок «Агрегация комплекта (a_tblAgg_List)»

**Зачем:** собрать **сводную отчётность** — суммировать данные дочерних организаций в формы родительской (головной) организации за период.

**Предварительно:** в **Редакторы → Агрегация** настроены связи parent ZID ↔ child ZID.

**Как пользоваться:**

1. Выберите **Сводная организация** (parent ZID).
2. Выберите **Период (EID)**.
3. Проверьте список участников свода.
4. Нажмите **Свести комплект**.

**Результат:** «Свод завершён: N форм…» — формы головной org заполнены суммой дочерних. При ошибках — панель увязок агрегации; исправьте данные участников и повторите.

---

### Блок «Агрегация вручную (один шаблон)»

**Зачем:** сложить в одну форму **несколько экземпляров одного шаблона** (например, разные филиалы или тестовая сводка).

**Как пользоваться:**

1. Выберите **Шаблон** (код формы).
2. Отметьте **минимум 2** экземпляра чекбоксами.
3. Нажмите **Создать агрегированную форму** (на кнопке отображается счётчик выбранных).

**Результат:** новый экземпляр с суммой выбранных; появится в **Мои формы**.

---

## 5. Редактор формы (дополнительно)

Все кнопки пользователя описаны во вкладке **«Пользователь организации»**, раздел «Редактор формы». У администратора есть дополнительная кнопка:

### Вернуть в черновик

**Зачем:** разрешить редактирование формы, которую пользователь уже **сдал**. Без этого правки невозможны.

**Когда:** пользователь обнаружил ошибку после сдачи или ЦО вернул форму на доработку.

**Как:**

1. **Мои формы** → найдите форму со статусом **Сдано** → **Открыть**.
2. Нажмите **Вернуть в черновик**.
3. Подтвердите в диалоге.

**Результат:** статус **Черновик**; жёлтая полоса исчезает; поля и кнопка **Сохранить** снова активны.

**Типовая цепочка:** Вернуть в черновик → правки → **Сохранить** → **Проверить форму** → **Сдать форму**.

---

## 6. Редакторы метаданных

**Меню → Редакторы → …**

### Зачем нужны редакторы

Здесь настраивается **логика системы**, а не данные отчётности: структура шаблонов, правила увязок, сальдо, Excel, расшифровки, агрегация. Изменения влияют на проверки и операции во всём портале.

> Редакторы доступны только при подключении к API (в углу экрана метка `POSTGRESQL`).

---

### Формы (`/admin/forms`)

**Назначение:** просмотр и правка метаданных шаблонов — названия, строки, колонки, типы ячеек.

**Как пользоваться:**

1. Кликните шаблон в списке слева.
2. Переключайте вкладки: метаданные, колонки, строки.
3. Внесите изменения → **Сохранить** на вкладке.

**Reimport** (в шапке): полная перезагрузка всех шаблонов из JSON-файлов на сервере. Используйте после обновления схем с диска; требует подтверждения.

---

### Увязки (`/admin/checks`)

**Назначение:** правила контрольных соотношений — что проверяет кнопка **Проверить форму** и **Проверить все формы**.

**Как пользоваться:**

1. Выберите строку в таблице или **Новая увязка**.
2. Заполните номер, выражение, флаги активности.
3. **Проверить на данных** — тест на реальных формах.
4. **Сохранить** — правило в базе.
5. **Удалить** — удаление правила.
6. **Reimport** — загрузка из `checks.json`.

---

### Сальдо (`/admin/saldo`)

**Назначение:** правила для блока **Перенести сальdo** на странице Сводка и импорт.

Две вкладки:

- **Правила сальдо** — соответствия ячеек при переносе (типы T/S/G).
- **FormCorrespondence** — соответствия форм для этапов Yellow/Red.

Кнопки: **Сохранить**, **Удалить**, **Reimport** на каждой вкладке.

---

### Excel (`/admin/excel`)

**Назначение:** настройка, как форма выгружается в Excel (кнопка **Excel** в редакторе формы).

Выберите или создайте запись маппинга → отредактируйте поля → **Сохранить**.

---

### Расшифровки (`/admin/rash`)

**Назначение:** пороги сумм и справочник правил для **Проверить расшифровки** (формы с контрагентами).

1. Вкладка порогов → **Сохранить пороги** (3 уровня).
2. Правила sp_rash → **Сохранить** / **Удалить**.
3. **Reimport** — из JSON.

---

### Агрегация (`/admin/aggregation`)

**Назначение:** кто входит в свод — связи parent ZID (головная org) и child ZID (участники).

1. Укажите parentZid, childZid, флаг **included**.
2. **Сохранить** / **Удалить**.
3. **Reimport** — из JSON.

Используется кнопкой **Свести комплект** в Сводка и импорт.

## 7. Пользователи

**Меню → Пользователи**

### Зачем нужен

Управление доступом: кто может войти в портал, с какой ролью и к какой организации привязан.

### Создать пользователя

1. Заполните **Логин**, **Пароль** (не короче 6 символов), **Имя**.
2. Выберите **Роль**: **Организация** или **Администратор**.
3. Для роли org — выберите **Организация** из списка ZID.
4. Нажмите **Создать**.

**Результат:** «Пользователь создан»; строка в таблице слева.

### Редактировать пользователя

1. Кликните строку в таблице.
2. Справа откроется панель **Редактирование: логин**.
3. Измените имя, пароль, роль, организацию, галочку **Учётная запись активна**.
4. **Сохранить** — изменения применены.
5. **Отмена** — закрыть без сохранения.

**Снять галочку «активна»** — пользователь не сможет войти (блокировка без удаления).

## 8. Комплекты (сводная таблица)

**Меню → Комплекты** (`/admin/packages`)

### Зачем нужен

Обзор **всей группы** на одном экране: по каждой паре «организация × период» видно % полноты, число черновиков и сданных форм. Удобно перед дедлайном сдачи.

### Как пользоваться

1. Откройте таблицу org × период.
2. Найдите ячейку с низким % или неполной сдачей.
3. Кликните **Открыть** или по ячейке — переход в **Комплект** с подставленными `zid` и `eid`.
4. Дальше: **Завести пустые формы**, контроль списка, связь с пользователем org.

**Результат:** быстрый переход к проблемному комплекту без ручного выбора ZID/EID.

## 9. Аудит

**Меню → Аудит**

### Зачем нужен

Журнал изменений **метаданных** (шаблоны, увязки, пользователи и т.д.): кто, когда и что менял. Не логирует ввод данных в ячейки форм.

### Как пользоваться

1. Строка **Поиск** — фильтр по тексту записи.
2. **← Назад** / **Вперёд →** — постраничная навигация (по 50 записей).
3. Читайте строки: время, пользователь, действие.

## 10. Типовые сценарии

### Новый отчётный период для группы

1. **Комплект** → для каждой org: **Создать период** (или общий EID).
2. **Завести пустые формы (комплект)** по каждой org.
3. **Пользователи** — убедиться, что у каждой org есть логин.
4. Сообщить пользователям: выбрать EID в Комплекте и начать заполнение.

### Контроль перед дедлайном

1. **Комплекты** — все ячейки 100% заведено.
2. **Сводка и импорт** → **Проверить все формы** (режим Период).
3. **Комплект** по каждой org — **Сдано** = 76 (или целевое число).
4. При необходимости — **Свести комплект** для головной org.

### Новая организация в группе

1. **Создать организацию** → ZID.
2. **Создать период** → EID.
3. **Завести пустые формы**.
4. **Пользователи → Создать** — логин для представителя org.
5. **Агрегация** — добавить child ZID в свод (если org входит в группу).

### Исправление сданной формы

1. **Мои формы** → форма **Сдано** → **Открыть**.
2. **Вернуть в черновик**.
3. Пользователь (или вы) вносит правки → **Сохранить** → **Проверить форму** → **Сдать форму**.

|--------|------------|-----------|

| **Экспорт JSON** | Резервная копия | Файл .json |
| **Импорт** | Загрузить JSON в форму | Данные заменены |

## Глоссарий

| Термин | Пояснение |
|--------|-----------|
| **ZID** | Код организации в справочнике |
| **EID** | Код отчётного периода |
| **Комплект** | До 76 экземпляров форм для пары ZID + EID |
| **Шаблон** | Описание структуры формы из каталога (76 штук) |
| **Экземпляр** | Заполненная копия шаблона с вашими данными |
| **Увязка** | Правило: значения в ячейках должны соотноситься определённым образом |
| **Сальдо** | Перенос остатков между формами/периодами |
| **Агрегация** | Суммирование данных нескольких org или экземпляров |
| **Черновик** | Форма в работе, редактирование открыто |
| **Сдано** | Форма зафиксирована, редактирование закрыто |
