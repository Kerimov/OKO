# Руководство по работе с порталом ОКО

Веб-портал ОКО предназначен для заполнения **76 форм** корпоративной (специализированной) отчётности. Данные хранятся на сервере (PostgreSQL на Render); портал открывается в браузере (Vercel).

В системе два типа учётных записей:

| Роль | Кому назначается | Основные возможности |
|------|------------------|----------------------|
| **Администратор** | Сотрудник ЦО / методолог | Все организации, редакторы правил, пользователи, сводные операции |
| **Пользователь организации** | Представитель конкретной организации | Только своя организация, заполнение и сдача форм |

---

# Часть 1. Инструкция администратора

## 1.1. Вход в систему

1. Откройте адрес портала в браузере (главная страница `/`).
2. Введите **логин** и **пароль**, выданные администратором системы.
3. Нажмите **Войти**.

После входа вы попадёте на страницу **Комплект** (`/package`) — рабочий центр по организациям и периодам.

> **Примечание.** Если в правом верхнем углу отображается метка БД (например, `POSTGRESQL · admin`), портал подключён к серверу. Без API портал работает только локально с данными в браузере.

### Выход

Внизу бокового меню нажмите **Выйти**. Сессия завершится, откроется страница входа.

---

## 1.2. Обзор интерфейса

Боковое меню состоит из разделов:

### Работа (доступно всем)
- **Каталог** — список из 76 шаблонов форм
- **Мои формы** — сохранённые заполненные экземпляры
- **Комплект** — выбор организации и периода, контроль полноты

### Операции (только администратор и legacy-токен)
- **Сводка и импорт** — проверки увязок, сальдо, агрегация, экспорт комплекта

### Редакторы (только администратор)
- **Формы** — конструктор шаблонов
- **Увязки** — правила контрольных соотношений
- **Сальдо** — перенос остатков между формами
- **Excel** — соответствие ячеек Excel и полей форм
- **Расшифровки** — справочник расшифровок контрагентов
- **Агрегация** — связи «сводная организация → дочерние»

### Администрирование (только администратор)
- **Комплекты** — сводная таблица полноты по всем организациям
- **Пользователи** — учётные записи для входа
- **Аудит** — журнал изменений метаданных

### Настройки
- Глобальные реквизиты по умолчанию (код предприятия, период, единица измерения)

---

## 1.3. Ключевые понятия

### ZID — идентификатор организации
Числовой код организации в справочнике `organizations`. К каждой организации привязаны периоды, формы и (опционально) пользователи.

### EID — идентификатор периода
Числовой код отчётного периода. Период всегда принадлежит конкретной организации (ZID).

### Комплект отчётности
Уникальная пара **ZID + EID** и набор форм (до 76 шаблонов) для этой организации за этот период. Аналог выбора организации и периода в MS Access.

### Рабочий контекст
Текущие ZID и EID, выбранные на странице **Комплект**. Новые формы из каталога автоматически привязываются к этому контексту.

### Статусы формы
| Статус | Значение |
|--------|----------|
| **Черновик** | Форма редактируется |
| **Сдано** | Форма зафиксирована; обычный пользователь не может её менять |

Администратор может вернуть сданную форму в черновик.

### Организация в справочнике vs поле в настройках
- **Справочник организаций** (ZID) — создаётся на странице **Комплект**, используется для комплектов и привязки пользователей.
- **Поле «Организация» в Настройках** — текст по умолчанию в шапке новых форм. Не заменяет справочник.

---

## 1.4. Первоначальная настройка

Рекомендуемый порядок при развёртывании новой системы:

### Шаг 1. Создать организации

1. Откройте **Комплект**.
2. В блоке **Добавить организацию** введите наименование.
3. Нажмите **Создать организацию**.
4. Повторите для всех организаций группы.

Система присвоит каждой организации уникальный ZID.

### Шаг 2. Создать отчётные периоды

1. На странице **Комплект** выберите организацию (ZID).
2. В блоке **Добавить период** укажите название (например, «1 квартал 2026»), при необходимости — даты начала и конца.
3. Нажмите **Создать период**.

Периоды создаются отдельно для каждой организации.

### Шаг 3. Завести комплект форм

1. Выберите организацию и период.
2. Нажмите **Завести пустые формы (комплект)**.

Будут созданы пустые экземпляры для всех шаблонов каталога (76 форм), кроме уже существующих. Прогресс-бар покажет полноту комплекта.

### Шаг 4. Создать пользователей организаций

1. Откройте **Администрирование → Пользователи**.
2. Заполните форму **Новый пользователь**:
   - **Логин** — уникальное имя для входа
   - **Пароль** — не менее 6 символов
   - **Отображаемое имя** — как показывать в интерфейсе
   - **Роль** — «Организация» или «Администратор»
   - **Организация** — для роли «Организация» обязательно выбрать ZID из справочника
3. Нажмите **Создать**.

### Шаг 5. Настроить глобальные реквизиты (опционально)

**Настройки →** укажите код предприятия, единицу измерения, период по умолчанию. Эти значения подставляются в новые формы, если не задан рабочий контекст ZID/EID.

---

## 1.5. Работа с комплектом отчётности

Страница **Комплект** — основной инструмент контроля.

### Выбор контекста
- Выпадающий список **Организация (ZID)**
- Выпадающий список **Период (EID)**

Выбор сохраняется автоматически и используется при создании новых форм.

### Контроль полноты
После выбора ZID и EID отображается:
- счётчик **заведено / всего** (из 76 шаблонов);
- количество черновиков и сданных форм;
- прогресс-бар;
- списки заведённых и незаведённых форм со ссылками.

### Сводная таблица по всем организациям
**Администрирование → Комплекты** — матрица «организация × период» с процентом полноты. Клик по ячейке открывает **Комплект** с нужными ZID и EID.

---

## 1.6. Заполнение форм

### Создание отдельной формы
1. **Каталог** → найдите форму по коду или названию.
2. Нажмите **Создать**.
3. Форма откроется в редакторе; экземпляр появится в **Мои формы**.

### Редактор формы (`/my/:id`)

**Верхняя панель:**
- редактирование названия экземпляра и реквизитов (организация, период, код предприятия);
- **Сохранить** — записать изменения на сервер;
- **Сдать** — зафиксировать форму (статус «Сдано»);
- **Вернуть в черновик** — только для администратора;
- **Удалить** — удалить экземпляр.

**Инструменты:**
- **Пересчитать** — применить правила итоговых строк;
- **Проверить увязки** — контрольные соотношения с другими формами;
- **Проверить расшифровки** — для форм контрагентов;
- **PDF** / **Excel** — выгрузка заполненной формы;
- **Экспорт JSON** / **Импорт JSON** — обмен данными;
- **Сбросить к шаблону** — очистить введённые данные.

Автопересчёт итоговых строк включается чекбоксом «Автопересчёт» (сохраняется в браузере).

### Список «Мои формы»
- Поиск и фильтры по типу формы и статусу.
- Чекбоксы и **Выбрать все** — массовое удаление.
- **Импорт JSON** — загрузка ранее экспортированного экземпляра.

---

## 1.7. Сводка и импорт (операции над комплектом)

Страница **Сводка и импорт** (`/tools`) — пакетные операции.

### Полнота комплекта
Показывает, сколько из 76 форм заведено в текущем рабочем контексте (или по датам из настроек).

### Сохранить комплект
- **JSON** — архив всех форм периода;
- **Excel** — выгрузка в табличный формат;
- **Пересчитать все формы** — массовый пересчёт итоговых строк.

### Проверка увязок
Режимы:
- **Период** — правила, актуальные для текущего периода;
- **Активные** — все активные правила;
- **Все** — полный набор (~3600 правил).

Результат: список нарушений с указанием форм и ячеек.

### Перенос сальдо
Перенос данных между формами по правилам `a_tblsaldo` или FormCorrespondence:
- выбор исходной и целевой формы;
- режим: по колонкам или детальный (T/S/G);
- фаза: предыдущий период / текущий.

### Агрегация
- **Агрегация комплекта** — сводная организация + дочерние ZID (настраивается в **Редакторы → Агрегация**);
- **Ручная агрегация** — объединение двух и более экземпляров одного шаблона.

---

## 1.8. Редакторы метаданных

Все редакторы требуют подключения к API-серверу.

### Формы (`/admin/forms`)
Редактирование шаблонов: метаданные, колонки, строки, превью таблицы. Кнопка **Reimport** — загрузка схем из JSON-файлов.

### Увязки (`/admin/checks`)
CRUD правил контрольных соотношений. Фильтры, пагинация, тест выражения. Флаги: active, periodActive, forAggrOnly.

### Сальдо (`/admin/saldo`)
Две вкладки: правила переноса остатков и соответствие форм (FormCorrespondence).

### Excel (`/admin/excel`)
Маппинг: какая ячейка Excel соответствует какому полю формы.

### Расшифровки (`/admin/rash`)
Справочник расшифровок для форм контрагентов; настройка порогов (3 уровня, тыс. руб.).

### Агрегация (`/admin/aggregation`)
Связи parentZid ↔ childZid для сводных комплектов; флаг включения в агрегацию.

> **Важно.** Изменения в редакторах влияют на все организации. Перед массовым reimport сделайте резервную копию БД.

---

## 1.9. Управление пользователями

**Администрирование → Пользователи**

### Создание
Форма вверху страницы: логин, пароль, имя, роль, организация (для роли «Организация»).

### Редактирование
1. Кликните строку в списке.
2. Справа откроется панель редактирования:
   - отображаемое имя;
   - новый пароль (оставьте пустым, чтобы не менять);
   - роль;
   - организация;
   - флаг «Учётная запись активна».
3. Нажмите **Сохранить**.

Логин после создания изменить нельзя. Отключённый пользователь не сможет войти.

---

## 1.10. Журнал аудита

**Администрирование → Аудит** — хронология изменений метаданных (кто, когда, что изменил). Поиск и постраничная навигация.

---

## 1.11. Типовые сценарии администратора

### Начало отчётного периода
1. Создать период (EID) для каждой организации.
2. Выбрать ZID + EID → **Завести пустые формы**.
3. Уведомить пользователей организаций о начале работы.

### Контроль перед сдачей
1. **Администрирование → Комплекты** — проверить полноту по всем org.
2. **Сводка и импорт** → проверка увязок в режиме «Период».
3. При необходимости — перенос сальдо.

### Исправление сданной формы
1. Открыть форму в **Мои формы**.
2. Нажать **Вернуть в черновик**.
3. Внести исправления → **Сохранить** → снова **Сдать**.

### Добавление новой организации в группу
1. **Комплект** → создать организацию.
2. Создать периоды.
3. Завести комплект.
4. **Пользователи** → создать учётную запись с привязкой к ZID.

# Часть 2. Инструкция пользователя организации

## 2.1. Вход в систему

1. Откройте адрес портала в браузере.
2. На главной странице введите **логин** и **пароль**, полученные от администратора.
3. Нажмите **Войти**.

После входа откроется раздел **Мои формы** — ваш личный кабинет.

Внизу бокового меню отображаются ваше имя и название организации.

### Выход
Нажмите **Выйти** внизу бокового меню.

---

## 2.2. Что доступно пользователю организации

### Доступно
| Раздел | Назначение |
|--------|------------|
| **Мои формы** | Ваши заполненные формы |
| **Комплект** | Полнота комплекта вашей организации |
| **Каталог** | Создание отдельных форм |
| **Настройки** | Реквизиты по умолчанию |

### Недоступно
- **Сводка и импорт** — пакетные проверки и агрегация
- Все разделы **Редакторы** и **Администрирование**
- Создание организаций и периодов
- Просмотр и редактирование форм других организаций
- Редактирование сданных форм

Вы видите и работаете **только с формами своей организации** (ваш ZID).

---

## 2.3. Начало работы

### Шаг 1. Проверьте комплект
1. Откройте **Комплект**.
2. Организация уже выбрана (изменить нельзя, если у вас одна организация).
3. Выберите **отчётный период** из списка.
4. Убедитесь, что комплект заведён (прогресс-бар показывает 76/76 или близко к этому).

Если форм не хватает — нажмите **Завести пустые формы (комплект)**. Будут созданы пустые экземпляры для недостающих шаблонов.

### Шаг 2. Откройте форму для заполнения
**Вариант А — из комплекта:**
- На странице **Комплект** в списке заведённых форм нажмите на нужную.

**Вариант Б — из «Мои формы»:**
- **Мои формы** → найдите форму по поиску или фильтру → **Открыть**.

**Вариант В — создать новую из каталога:**
- **Каталог** → найдите шаблон → **Создать**.

---

## 2.4. Заполнение формы

### Реквизиты
В верхней части формы проверьте:
- **Организация** — должна соответствовать вашей;
- **Период** — начало и конец отчётного периода;
- **Код предприятия**, **единица измерения**.

### Таблица
- Вводите данные в ячейки таблицы (как в оригинальных PDF-формах).
- Итоговые строки могут пересчитываться автоматически (если включён **Автопересчёт**).
- Для ручного пересчёта нажмите **Пересчитать**.

### Сохранение
Нажимайте **Сохранить** регулярно — данные записываются на сервер. Несохранённые изменения при закрытии вкладки могут быть потеряны.

### Проверка перед сдачей
- **Проверить увязки** — покажет ошибки в соотношениях между формами;
- **Проверить расшифровки** — для форм с контрагентами.

Исправьте все ошибки (красные) перед сдачей.

### Сдача формы
1. Убедитесь, что форма сохранена.
2. Нажмите **Сдать**.
3. Подтвердите действие.

После сдачи форма переходит в статус **«Сдано»** и **больше не редактируется**. Для исправлений обратитесь к администратору — он вернёт форму в черновик.

### Выгрузка
- **PDF** — печатная версия формы;
- **Excel** — табличная выгрузка;
- **Экспорт JSON** — резервная копия данных формы.

---

## 2.5. Раздел «Мои формы»

Список всех ваших сохранённых форм.

### Поиск и фильтры
- Поиск по названию, коду формы, организации.
- Фильтр по типу формы (N01_1, N02_1 и т.д.).
- Фильтр по статусу: черновики / сдано.

### Удаление
- **Одна форма** — кнопка **Удалить** на карточке.
- **Несколько форм** — отметьте чекбоксами → **Удалить выбранные**.

> Удаление безвозвратно. Не удаляйте сданные формы без согласования с администратором.

### Импорт
Кнопка **Импорт JSON** — загрузка ранее экспортированного экземпляра (например, после работы офлайн).

---

## 2.6. Раздел «Комплект»

Ваш основной инструмент контроля готовности отчётности.

1. **Выберите период** — переключайте EID при смене отчётного периода.
2. **Смотрите прогресс** — сколько форм заведено, сколько в черновике, сколько сдано.
3. **Незаведённые формы** — список шаблонов, для которых ещё нет экземпляра; ссылка ведёт в каталог.
4. **Завести пустые формы** — создать недостающие экземпляры одной кнопкой.

Рабочий контекст (ZID + EID) запоминается: новые формы из каталога автоматически привязываются к выбранному периоду.

---

## 2.7. Раздел «Каталог»

Полный список из 76 шаблонов форм корпоративной отчётности.

- **Поиск** по коду (например, `N01_1`) или названию.
- **Фильтр по разделу** (N01, N02, … ND).
- **Создать** — новый экземпляр формы в «Мои формы».
- **PDF** — просмотр образца бланка (если загружен на сервер).

Используйте каталог, когда нужна одна конкретная форма вне пакетного заведения комплекта.

---

## 2.8. Настройки

**Настройки** — значения по умолчанию для **новых** форм:
- код предприятия;
- текстовое поле «Организация»;
- даты периода;
- единица измерения (обычно «тыс.руб.»).

Нажмите **Сохранить настройки** после изменений.

> Это не справочник организаций. Ваша привязка к организации (ZID) задаётся администратором в учётной записи.

В блоке **Учётная запись** отображаются ваш логин, роль и организация; кнопка **Выйти**.

## 2.9. Типовые сценарии пользователя

### Ежедневная работа
1. Войти → **Мои формы**.
2. Открыть форму со статусом «Черновик».
3. Заполнить → **Сохранить**.
4. Перейти к следующей форме.

### Завершение отчётного периода
1. **Комплект** → выбрать период → убедиться, что все 76 форм заведены.
2. Заполнить оставшиеся черновики.
3. По каждой форме: **Проверить увязки** → исправить → **Сдать**.
4. **Комплект** → убедиться, что все формы в статусе «Сдано».

### Ошибка в сданной форме
Обратитесь к администратору. Самостоятельно сданную форму изменить нельзя.

### Работа с новым периодом
1. **Комплект** → выберите новый период (EID).
2. **Завести пустые формы** — если комплект ещё не создан.
3. Заполняйте формы как обычно.

## 2.10. Частые вопросы (пользователь)

**Почему я не вижу раздел «Сводка и импорт»?**  
Этот раздел доступен только администраторам. Пакетные проверки выполняет ЦО.

**Почему не могу выбрать другую организацию?**  
Ваша учётная запись привязана к одной организации. Это сделано для разграничения данных.

**Форма не сохраняется / ошибка доступа**  
Проверьте подключение к интернету. Если форма в статусе «Сдано» — редактирование заблокировано. Обратитесь к администратору.

**Куда делись мои данные после смены браузера?**  
Данные хранятся на сервере, не в браузере. Войдите под тем же логином с любого устройства.

**Что означают статусы?**  
- **Черновик** — можно редактировать.  
- **Сдано** — форма зафиксирована, ожидает приёмки.
