# Инструкция администратора

Портал ОКО — система корпоративной отчётности для группы организаций. Администратор настраивает справочники, комплекты, правила проверок, учётные записи и контролирует полноту сдачи отчётности.

## Введение

### Назначение роли администратора

Администратор (ЦО, методолог) обеспечивает:

- справочник организаций и периодов;
- заведение комплектов форм;
- контроль полноты и увязок по всей группе;
- настройку правил (увязки, сальдо, Excel, расшифровки, агрегация);
- учётные записи пользователей организаций;
- возврат сданных форм в черновик при исправлениях.

### Отличие от пользователя организации

| Возможность | Администратор | Пользователь org |
|-------------|:-------------:|:----------------:|
| Все организации (ZID) | ✓ | только своя |
| Создание org / period | ✓ | ✗ |
| Сводка и импорт | ✓ | ✗ |
| Редакторы метаданных | ✓ | ✗ |
| Управление пользователями | ✓ | ✗ |
| Возврат «Сдано» → черновик | ✓ | ✗ |

## Вход и интерфейс

### Вход

1. Главная страница `/` → логин и пароль.
2. **Войти**.

**Результат:** переход на **Комплект** (`/package`).

### Структура меню

**Работа:** Каталог, Мои формы, Комплект.

**Операции:** Сводка и импорт — пакетные проверки, сальдо, агрегация, экспорт.

**Редакторы:** Формы, Увязки, Сальдо, Excel, Расшифровки, Агрегация.

**Администрирование:** Комплекты (dashboard), Пользователи, Аудит.

**Прочее:** Инструкция, Настройки.

### Индикатор БД

Метка `POSTGRESQL · admin` (или `sqlite`) — подключение к API и тип базы данных.

## Ключевые понятия

### ZID и EID

- **ZID** — ID организации в таблице `organizations`.
- **EID** — ID периода в `periods` (привязан к ZID).

### Комплект

Пара ZID+EID + до 76 экземпляров форм (по шаблонам каталога).

### Рабочий контекст

Текущие `workZid` и `workEid` в настройках сервера. Используются при создании форм и в **Сводка и импорт**.

### Справочник org vs Настройки

| | Справочник (`/package`) | Настройки (`/settings`) |
|--|-------------------------|-------------------------|
| Назначение | Юрлица для комплектов, ZID | Текст по умолчанию в шапке |
| Кто меняет | Администратор | Любой пользователь |
| Влияние | Привязка форм, пользователей | Только новые формы без контекста |

### Статусы

- **Черновик** — редактируется всеми с доступом.
- **Сдано** — пользователь org не редактирует; администратор может **Вернуть в черновик**.

## Первоначальная настройка системы

### Шаг 1. Создать организации

**Где:** **Комплект** → **Добавить организацию**.

**Действия:** ввести наименование → **Создать организацию**.

**Результат:** новый ZID, организация в списке селектора.

### Шаг 2. Создать периоды

**Где:** **Комплект** → выбрать ZID → **Добавить период**.

**Поля:** название, даты начала/конца (опционально).

**Результат:** новый EID для выбранной организации.

### Шаг 3. Завести комплекты

**Действия:** ZID + EID → **Завести пустые формы (комплект)**.

**Результат:** до 76 пустых экземпляров на org×period.

### Шаг 4. Создать пользователей

**Где:** **Пользователи** → форма **Новый пользователь**.

**Поля:** логин, пароль (≥6), имя, роль, организация (для org).

**Результат:** пользователь может войти и видеть только свою org.

### Шаг 5. Глобальные настройки

**Настройки** → код предприятия, единица, период по умолчанию.

## Раздел «Комплект»

Путь: `/package`.

### Рабочий контекст

Селекторы ZID и EID для любой организации группы. Выбор сохраняется и используется при создании форм.

**Результат:** блок полноты и кнопка заведения комплекта активны.

### Добавить организацию / период

Только администратор. См. [Первоначальная настройка](#первоначальная-настройка-системы).

### Полнота комплекта

| Показатель | Смысл |
|------------|-------|
| `N/76` | Сколько шаблонов имеют экземпляр |
| Черновики | Формы в работе |
| Сдано | Зафиксированные формы |
| Прогресс-бар | Доля заведённых шаблонов |

**Завести пустые формы:** создаёт недостающие экземпляры, не трогает существующие.

**Списки:** заведённые (со ссылками и статусами), незаведённые (ссылки в каталог).

## Раздел «Мои формы» и редактор формы

Администратор видит формы в контексте выбранной работы (или все, если нет org-ограничения на уровне API).

### Мои формы

Поиск, фильтры, массовое удаление, импорт JSON — как у пользователя org, но без ограничения одной организацией (в зависимости от учётной записи).

### Редактор формы — дополнительно для admin

| Функция | Назначение | Результат |
|---------|------------|-----------|
| **Вернуть в черновик** | Разблокировать сданную форму | Статус «Черновик», пользователь может править |
| Все остальные кнопки | См. инструкцию пользователя | — |

**Проверить форму** — увязки по всем формам, доступным в контексте проверки.

**Сдать форму** — фиксация от имени организации-владельца формы.

## Раздел «Сводка и импорт»

Путь: `/tools`. Пакетные операции над комплектом.

### Полнота комплекта

**Назначение:** обзор заведённых форм по текущему work-context или датам из настроек.

**Результат:** счётчик, список отсутствующих шаблонов.

### Сохранить комплект

| Кнопка | Назначение | Результат |
|--------|------------|-----------|
| **JSON** | Архив всех экземпляров периода | Файл `.json` для архива/переноса |
| **Excel** | Табличная выгрузка комплекта | Файл `.xlsx` |
| **Пересчитать все формы** | Массовый пересчёт итоговых строк | Обновлённые значения во всех формах периода |

### Проверка увязок

**Режимы:**

| Режим | Что проверяется |
|-------|-----------------|
| **Период** | Правила, актуальные для текущего периода |
| **Активные** | Все активные правила |
| **Все** | Полный набор (~3600 правил) |

**Действие:** **Проверить увязки**.

**Результат:** панель с нарушениями (номер правила, формы, ячейки, выражения) или «все проверки пройдены».

### Перенос сальдо

**Назначение:** перенос остатков между формами по правилам `a_tblsaldo` или FormCorrespondence.

**Параметры:**

- **Исходная / целевая форма** — экземпляры из списка;
- **Фаза** — предыдущий период / текущий;
- **Режим** — по колонкам или детальный (T/S/G).

**Результат:** значения в целевой форме обновлены, форма сохранена на сервере.

### Агрегация комплекта

**Назначение:** сведение данных дочерних организаций в формы сводной (parent ZID).

**Предусловие:** настроены связи в **Редакторы → Агрегация**.

**Параметры:** сводная организация (parent ZID), EID, список дочерних ZID.

**Действие:** **Выполнить агрегацию комплекта**.

**Результат:** сообщение о количестве агрегированных форм; опционально проверка увязок агрегации.

### Ручная агрегация

**Назначение:** объединить 2+ экземпляра **одного шаблона** (например, ручное сложение).

**Результат:** новый или обновлённый экземпляр с суммированными значениями.

## Редактор «Формы»

Путь: `/admin/forms`.

### Назначение

Редактирование **шаблонов** форм в базе (метаданные, колонки, строки). Влияет на все новые экземпляры.

### Как работает

1. Список шаблонов слева.
2. Вкладки: метаданные, колонки, строки, превью.
3. **Сохранить** — запись в БД.
4. **Reimport** — перезагрузка всех схем из JSON-файлов (осторожно: перезапись).

**Результат:** изменённая структура таблицы при создании новых форм.

---

## Редактор «Увязки»

Путь: `/admin/checks`.

### Назначение

Правила контрольных соотношений (`a_tblchecks`) — формулы, связывающие ячейки разных форм.

### Элементы правила

| Поле | Смысл |
|------|-------|
| Номер | Уникальный ID правила |
| Выражение | Формула проверки |
| Флаги active / periodActive / forAggrOnly | Когда правило участвует в проверке |

### Действия

- Создать / редактировать / удалить правило.
- **Тест выражения** — проверка синтаксиса на текущих данных.
- **Reimport** — загрузка из `checks.json`.

**Результат:** кнопка **Проверить форму** и **Сводка и импорт** используют обновлённые правила.

---

## Редактор «Сальдо»

Путь: `/admin/saldo`.

### Вкладка «Правила сальдо»

Правила переноса остатков (T/S/G) между формами и ячейками.

**Результат:** используется в **Сводка и импорт → Перенос сальдо**.

### Вкладка «FormCorrespondence»

Соответствие форм для пакетного переноса.

**Результат:** альтернативный режим переноса сальдо.

---

## Редактор «Excel»

Путь: `/admin/excel`.

### Назначение

Маппинг: ячейка Excel ↔ поле формы (formName, sheetName, excelRow/Column, formColumn/Row).

**Результат:** кнопка **Excel** в редакторе формы и экспорт комплекта формируют файлы по этим правилам.

## Редактор «Расшифровки»

Путь: `/admin/rash`.

### Назначение

Справочник `sp_rash` для форм контрагентов; пороги сумм (3 уровня, тыс. руб.).

**Результат:** **Проверить расшифровки** в редакторе форм контрагентов.

## Редактор «Агрегация»

Путь: `/admin/aggregation`.

### Назначение

Связи **parentZid → childZid** для сводных комплектов; флаг **included**.

**Результат:** **Сводка и импорт → Агрегация комплекта** знает, какие дочерние org входят в свод.

## Администрирование «Комплекты»

Путь: `/admin/packages`.

### Назначение

Сводная матрица: все организации × все периоды.

**Колонки:** полнота %, черновики, сдано.

**Действие:** клик по ячейке → **Комплект** с нужными `?zid=&eid=`.

**Результат:** быстрый контроль готовности группы перед сдачей.

---

## Администрирование «Пользователи»

Путь: `/admin/users`.

### Создание пользователя

Форма вверху: логин, пароль, имя, роль (admin/org), организация.

**Результат:** учётная запись в БД, пользователь может войти.

### Редактирование

1. Клик по строке в списке.
2. Панель справа: имя, пароль (пусто = не менять), роль, org, активность.
3. **Сохранить**.

**Результат:** обновлённый профиль; неактивный пользователь не войдёт.

**Ограничение:** логин после создания не меняется.

---

## Администрирование «Аудит»

Путь: `/admin/audit`.

### Назначение

Журнал `report_log`: кто, когда и что изменил в метаданных (редакторы форм, увязок и т.д.).

**Результат:** поиск по тексту, постраничный просмотр для расследования изменений.

## Раздел «Настройки»

Глобальные реквизиты, блок учётной записи, **Выйти**.

При **legacy-токене** (Bearer без профиля): предупреждение и **Сбросить токен** → вход по логину для полного функционала.

## Типовые сценарии администратора

### Старт отчётного периода

1. Создать EID для каждой org (или нужных org).
2. Завести комплекты (ZID+EID).
3. Создать/проверить пользователей org.
4. Уведомить организации о начале работы.

**Итог:** все org могут заполнять формы за новый период.

### Контроль перед приёмкой

1. **Комплекты** — 100% заведения по всем org.
2. **Сводка и импорт** — увязки режим «Период».
3. Перенос сальдо при необходимости.
4. Агрегация сводной org.

**Итог:** выявлены и исправлены нарушения до дедлайна.

### Исправление сданной формы

1. **Мои формы** → открыть форму.
2. **Вернуть в черновик**.
3. Правки → **Сохранить** → уведомить пользователя или **Сдать** снова.

**Итог:** форма исправлена и снова зафиксирована.

### Новая организация в группе

1. **Комплект** → создать org.
2. Периоды → завести комплект.
3. **Пользователи** → учётная запись org + ZID.
4. **Агрегация** — при необходимости добавить child ZID к parent.

**Итог:** новое юрлицо в контуре отчётности.

## Введение

### Назначение

Портал заменяет работу в MS Access «ПК ОКО»: вы заполняете табличные формы отчётности, сохраняете их на сервере, проверяете увязки и **сдаёте** готовые формы.

### Ваша роль

Учётная запись с ролью **«Организация»** привязана к одному юрлицу (коду **ZID**). Все созданные и сохранённые формы автоматически относятся к вашей организации.

### Что вы можете делать

| Действие | Результат |
|----------|-----------|
| Заводить комплект форм на период | Появляются пустые экземпляры всех 76 шаблонов |
| Заполнять и сохранять формы | Данные хранятся на сервере, доступны с любого компьютера |
| Проверять увязки | Система показывает ошибки в соотношениях между формами |
| Сдавать формы | Форма фиксируется и не редактируется без администратора |
| Выгружать PDF / Excel / JSON | Файл для печати, анализа или резервной копии |

### Что недоступно

- Раздел **Сводка и импорт** (пакетные операции ЦО)
- Редакторы правил, пользователей, аудит
- Создание организаций и периодов в справочнике
- Просмотр и изменение чужих форм
- Редактирование форм в статусе **«Сдано»**

## Вход и выход

### Назначение

Авторизация по логину и паролю защищает данные и определяет, к какой организации вы привязаны.

### Как войти

1. Откройте адрес портала в браузере (главная страница).
2. Введите **логин** и **пароль**, выданные администратором.
3. Нажмите **Войти**.

**Результат:** откроется раздел **Мои формы**. Внизу бокового меню — ваше имя и название организации.

### Как выйти

Внизу бокового меню нажмите **Выйти**.

**Результат:** сессия завершена, отображается страница входа. Для продолжения работы войдите снова.

| **Настройки** | `/settings` | Реквизиты по умолчанию, выход из учётной записи |

### Индикатор в углу экрана

Если видна метка вида `POSTGRESQL · user` — портал подключён к серверу, данные сохраняются в базе.

## Ключевые понятия

### ZID — организация

Числовой код вашего юрлица в справочнике. Задаётся администратором. Все ваши формы имеют этот ZID.

### EID — отчётный период

Числовой код периода (квартал, год и т.д.) в рамках вашей организации. Один период — один комплект из до 76 форм.

### Комплект отчётности

Набор форм для пары **ZID + EID**. Полный комплект — **76 экземпляров** (по одному на каждый шаблон каталога).

### Рабочий контекст

Текущие ZID и EID, выбранные на странице **Комплект**. Новые формы из каталога автоматически привязываются к этому периоду.

### Статусы формы

| Статус | Смысл | Что можно делать |
|--------|-------|------------------|
| **Черновик** | Форма в работе | Редактировать, сохранять, удалять, сдавать |
| **Сдано** | Форма зафиксирована | Только просмотр, PDF/Excel/JSON; правки — через администратора |

### Шаблон и экземпляр

- **Шаблон** — тип формы в каталоге (например, `N01_1`), общий для всех организаций.
- **Экземпляр** — ваша конкретная заполненная (или пустая) форма с уникальным ID, привязанная к ZID, EID и периоду.

## Раздел «Комплект»

Путь: **Комплект** в меню (`/package`).

### Назначение

Контроль готовности отчётности за выбранный период: сколько форм заведено, сколько заполнено и сдано. Заведение пустого комплекта одной кнопкой.

### Рабочий контекст

**Элементы:** выпадающие списки **Организация (ZID)** и **Период (EID)**.

**Как пользоваться:**

1. Откройте **Комплект**.
2. Организация обычно уже выбрана (изменить нельзя, если организация одна).
3. Выберите **период** из списка EID.

**Как работает:** выбор сохраняется на сервере. Все новые формы из каталога получают текущий ZID и EID.

**Результат:** под селекторами отображается название организации и даты периода.

### Полнота комплекта

**Элементы:** счётчик `заведено / 76`, число черновиков и сданных, прогресс-бар, списки форм.

**Как работает:** система сравнивает каталог из 76 шаблонов с экземплярами вашей организации за выбранный EID.

**Результат:**

- **76/76** — все шаблоны заведены (могут быть пустыми или заполненными).
- Меньше 76 — в блоке **«Не заведено»** список недостающих форм со ссылками в каталог.

### Завести пустые формы (комплект)

**Кнопка:** **Завести пустые формы (комплект)**.

**Как пользоваться:**

1. Выберите ZID и EID.
2. Нажмите кнопку.
3. Дождитесь сообщения о результате.

**Как работает:** для каждого шаблона каталога, у которого ещё нет экземпляра в этом комплекте, создаётся пустая форма с предзаполненными строками показателей из шаблона.

**Результат:** сообщение вида «создано N, пропущено M»; прогресс-бар обновляется; формы появляются в **Мои формы** и в списке **«Заведено»**.

### Списки заведённых и незаведённых форм

- **Заведено** — раскрывающийся список с кодом формы, названием, статусом; клик открывает редактор.
- **Не заведено** — шаблоны без экземпляра; ссылка ведёт в **Каталог** для ручного создания.

---

## Раздел «Мои формы»

Путь: **Мои формы** (`/my`).

### Назначение

Единый список всех ваших экземпляров форм: поиск, фильтрация, массовое удаление, импорт.

### Поиск и фильтры

| Элемент | Назначение |
|---------|------------|
| Строка поиска | Поиск по названию, коду формы, организации |
| **Все типы форм** | Фильтр по коду шаблона (N01_1, N02_1…) |
| **Все статусы** | Черновики / Сдано |

**Результат:** список карточек сокращается до подходящих записей.

### Карточка формы

На карточке отображаются: название, код шаблона, полное название формы, статус, организация, период, даты создания и изменения.

| Кнопка | Действие | Результат |
|--------|----------|-----------|
| **Открыть** / клик по названию | Редактор формы | Переход в `/my/:id` |
| **Удалить** | Удаление одной формы | После подтверждения форма удалена безвозвратно |

### Массовое удаление

1. Отметьте чекбоксы у нужных форм или **Выбрать все** (с учётом фильтра).
2. Нажмите **Удалить выбранные (N)**.
3. Подтвердите.

**Результат:** выбранные экземпляры удалены из базы.

### Импорт JSON

**Кнопка:** **Импорт JSON**.

**Назначение:** загрузить ранее экспортированный экземпляр (резервная копия или перенос).

**Результат:** новый экземпляр в списке, открывается редактор.

---

## Раздел «Каталог»

Путь: **Каталог** (`/catalog`).

### Назначение

Просмотр всех 76 шаблонов и создание отдельного экземпляра вне пакетного заведения комплекта.

### Поиск и фильтр по разделу

- Поиск по коду (`N01_1`) или названию.
- Фильтр **Все разделы** / N01, N02 … ND.

**Результат:** отфильтрованная сетка карточек шаблонов.

### Карточка шаблона

| Элемент | Действие | Результат |
|---------|----------|-----------|
| **Создать** | Новый экземпляр | Форма создаётся с ZID/EID из рабочего контекста, открывается редактор |
| **PDF** | Образец бланка | Открывается PDF-образец в новой вкладке (если загружен на сервер) |

**Важно:** перед созданием проверьте рабочий контекст на странице **Комплект**, иначе форма может попасть не в тот период.

## Редактор формы

Путь: `/my/:instanceId` (из **Мои формы** или **Комплект**).

### Назначение

Ввод данных в таблицу формы, проверки, сохранение, сдача, выгрузка.

### Реквизиты формы

В верхней части:

| Поле | Назначение |
|------|------------|
| **Название сохранённой формы** | Произвольное имя для списка «Мои формы» |
| **Организация** | Наименование юрлица в шапке формы |
| **Код предприятия** | Код в формате ОКО (например, `1@1`) |
| **Начало / конец периода** | Даты отчётного периода |
| **Единица измерения** | Обычно «тыс.руб.» |
| **Подписи** | ФИО и должности (если предусмотрены шаблоном) |

Изменения в реквизитах сохраняются при **Сохранить** или при потере фокуса с названия.

### Таблица данных

**Как работает:** строки и колонки соответствуют бланку PDF. Часть ячеек — только для чтения (итоги, коды строк). Числовые поля участвуют в пересчёте и увязках.

**Результат ввода:** значения хранятся в экземпляре до нажатия **Сохранить**.

### Кнопки панели инструментов

#### Сохранить

**Назначение:** записать все изменения на сервер.

**Результат:** статус «Сохранено» с временем; данные не потеряются при закрытии браузера.

#### Сдать форму

**Назначение:** зафиксировать форму как готовую к приёмке.

**Как пользоваться:** сохраните форму → **Сдать форму** → подтвердите.

**Результат:** статус **«Сдано»**; редактирование заблокировано; появляется предупреждение о режиме только просмотра.

#### Пересчёт

**Назначение:** пересчитать итоговые строки и графы по правилам шаблона.

**Результат:** обновлённые значения в итоговых ячейках; при включённом **Автопересчёте** выполняется автоматически через ~0,5 с после ввода.

#### Автопересчёт (чекбокс)

**Назначение:** автоматический пересчёт при изменении ячеек.

**Результат:** настройка сохраняется в браузере.

#### Проверить форму

**Назначение:** контрольные соотношения (увязки) между этой и другими формами комплекта.

**Как работает:** система загружает правила из базы, сравнивает значения ячеек.

**Результат:** панель с списком нарушений (форма, ячейка, ожидаемое/фактическое) или сообщение «Проверка пройдена».

#### Проверить расшифровки

**Назначение:** только для форм контрагентов — контроль расшифровок по справочнику `sp_rash`.

**Результат:** список ошибок и предупреждений по порогам сумм.

#### Сохранить PDF

**Назначение:** выгрузка заполненной формы в PDF с кириллицей, таблицей и реквизитами.

**Результат:** файл PDF скачивается в папку загрузок браузера.

#### Excel

**Назначение:** выгрузка данных формы в Excel по маппингу ячеек.

**Результат:** файл `.xlsx` для анализа в Excel.

#### Экспорт JSON

**Назначение:** полная резервная копия экземпляра (все ячейки, реквизиты, метаданные).

**Результат:** файл `.json`, который можно импортировать через **Мои формы → Импорт JSON**.

#### Импорт

**Назначение:** заменить данные текущего экземпляра из JSON-файла (осторожно: перезапись).

#### Сбросить данные

**Назначение:** очистить введённые значения, вернуть строки к шаблону.

**Результат:** пустая (шаблонная) форма; требуется подтверждение.

#### Удалить

**Назначение:** удалить экземпляр безвозвратно.

**Результат:** переход в **Мои формы**.

#### Образец PDF

**Назначение:** открыть пустой бланк формы для сверки структуры.

### Режим «Сдано»

Если форма сдана: поля и кнопка **Сохранить** недоступны, доступны просмотр и выгрузка PDF/Excel/JSON. Для исправлений обратитесь к администратору.

## Раздел «Настройки»

Путь: **Настройки** (`/settings`).

### Назначение

Значения по умолчанию для **новых** форм (не меняет уже созданные экземпляры).

| Поле | Назначение |
|------|------------|
| Код предприятия | Подставляется в новые формы |
| Организация (текст) | Текст в шапке; **не** заменяет справочник ZID |
| Начало / конец периода | Fallback, если не задан EID |
| Единица измерения | Например, «тыс.руб.» |

**Кнопка:** **Сохранить настройки**.

**Результат:** следующие созданные формы получат эти реквизиты (плюс данные из рабочего контекста ZID/EID).

### Учётная запись

Отображаются логин, роль, организация. Кнопка **Выйти**.

## Типовые сценарии

### Начало работы в новом периоде

1. **Комплект** → выберите новый EID.
2. **Завести пустые формы (комплект)**.
3. **Мои формы** → открывайте черновики по очереди.
4. Заполняйте → **Сохранить** → **Проверить форму** → **Сдать форму**.
5. **Комплект** → убедитесь: 76/76 заведено, все нужные формы в статусе «Сдано».

**Итог:** комплект за период заполнен и сдан.

### Ежедневная работа

1. Вход → **Мои формы**.
2. Фильтр **Черновики**.
3. Открыть → заполнить → **Сохранить**.
4. Перейти к следующей форме.

**Итог:** прогресс сохраняется на сервере после каждого **Сохранить**.

### Ошибка в сданной форме

1. Свяжитесь с администратором.
2. Администратор вернёт форму в черновик.
3. Вы вносите правки → **Сохранить** → **Сдать** снова.

**Итог:** исправленная форма снова в статусе «Сдано».

---

## Частые вопросы

**Почему нет «Сводка и импорт»?**  
Раздел только для администраторов ЦО.

**Почему не могу сменить организацию?**  
Учётная запись привязана к одному ZID.

**Форма не сохраняется**  
Проверьте интернет. Если статус «Сдано» — редактирование закрыто.

**Данные на другом компьютере**  
Войдите под тем же логином — данные на сервере.

**Чем отличается «Сохранить» от «Сдать»?**  
**Сохранить** — промежуточная запись. **Сдать** — финальная фиксация без права правки.
