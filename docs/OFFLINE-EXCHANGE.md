# Офлайн-обмен с дочерними организациями

Временный канал для филиалов **без доступа** к веб-порталу ЦО. Аналог обмена файлами Access (папка 12345), но в формате OKO JSON.

---

## Схема

```
ЦО (портал + PostgreSQL)
  │  1. Завести комплект org + период
  │  2. Выгрузить JSON / собрать offline-kit
  ▼
Дочка (OKO Offline на ПК)
  │  3. Заполнить формы локально
  │  4. Экспорт JSON комплекта
  ▼
ЦО
  5. Сводка и импорт → Импорт комплекта JSON
  6. Проверить все формы, агрегация
```

---

## Для центрального офиса (admin)

### Подготовка комплекта

1. **Комплект** → выберите организацию дочки (ZID) и период (EID).
2. **Завести пустые формы (комплект)** — 76 форм.
3. **Сводка и импорт** → блок **«Обмен с дочерними организациями»**.

### Выгрузка для дочки

- **Выгрузить комплект для дочки** — скачивается `oko_package_….json`.
- Передайте файл дочке **или** соберите offline-kit (см. ниже).

### Приём заполненного комплекта

1. Убедитесь, что в **Комплект** выбраны тот же ZID и EID.
2. **Импорт комплекта JSON** → выберите файл от дочки.
3. Галочка **«Перезаписать существующие формы»** — обычно включена.
4. **Проверить все формы** — контроль увязок.

## Сборка offline-kit (ZIP для дочки)

Дочке не нужен доступ к Render/Vercel — только распакованная папка и Node.js.

```bash
# Из корня репозитория, после выгрузки JSON из портала:
chmod +x scripts/build-offline-kit.sh
./scripts/build-offline-kit.sh ./oko_package_Дочка_20260331.json
```

Результат: **`dist/oko-offline-kit.zip`**

Содержимое:
- статический портал (офлайн-режим);
- предзагруженный комплект `offline-package.json`;
- `start.bat` / `start.sh` — локальный сервер на порту 8787;
- `README.txt` — инструкция для пользователя.

Передайте zip дочке. После заполнения дочка присылает **один JSON** обратно.

---

## Для дочерней организации

1. Распаковать `oko-offline-kit.zip`.
2. Запустить `start.bat` (Windows) или `start.sh` (macOS/Linux).
3. Браузер: http://localhost:8787
4. Заполнить формы в **Мои формы**.
5. **Отправить в ЦО** → **Сохранить комплект JSON**.
6. Отправить файл в ЦО.

В меню только: Мои формы, Каталог, Отправить в ЦО. Вход и сервер не требуются.

## Формат файла комплекта

JSON версии 1.1:

```json
{
  "version": "1.1",
  "organization": "ООО Дочка",
  "periodStart": "2026-01-01",
  "periodEnd": "2026-03-31",
  "zid": 2,
  "eid": 5,
  "instanceCount": 76,
  "instances": [ … ]
}
```

---

## API (для интеграций)

`POST /api/packages/import` (admin)

```json
{
  "zid": 2,
  "eid": 5,
  "overwrite": true,
  "package": {
    "organization": "ООО Дочка",
    "instances": [ … ]
  }
}
```

Ответ: `{ "created", "updated", "skipped", "errors" }`.
